For the while our API is not limited and not require any key. Later on, when we will enable access to historical data, things may change.

{{ _("Routing") }}

{{ _("Returns polyline objects (a Google standard format) and navigation comments based on a user query. There are 3 modes: ") }}

takeBike

dropBike

fullTrip

{{ _("These are illustrated in the following examples") }}. The idea is that a user will always want to do one of three things (pick a bike up, drop a bike off or pick up bike and drop it off somewhere else). The other parameters are the departure and the arrival (in [lat, lon] format), the time of departure (in epoch milliseconds, which can be in the past) and the number of people. In the case of solely picking up a bike or dropping one off, the departure and the arrival have to be equal to perform the action at one point, although this is redundant it allows more flexibility. In the case of a full trip the departure and the arrival should be different, the object returned will be a list of successive paths to do by foot or by bike.
  • openbikes.co/api/routing/mode=dropBike&city=New-York&departure=[40.76727,-73.99392]&arrival=[40.71911,-74.0066]&time=[1442935490422]&people=3
  • openbikes.co/api/routing/mode=takeBike&city=Toulouse&departure=[43.5639677,1.4655774]&arrival=[43.6044328,1.443463]&time=[1442935490355]&people=1
  • openbikes.co/api/routing/mode=fullTrip&city=Dublin&departure=[53.34956,-6.27819]&arrival=[53.33412,-6.26543]&time=[1442935491010]&people=2
