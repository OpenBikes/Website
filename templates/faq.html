Frequently Asked Questions









{{ _("Why is my city not on the main page?") }}

{{ _("There are a lot of cities who provide bike sharing. However, not all of these cities have linked their bike hubs to a central computer, hence we do not have access to their data.") }}

{{ _("Why have you built OpenBikes when similar applications already exist?") }}

{{ _("We believe that we compare well to other applications because we are honest about the data we are using while they are not. For example, there have already been applications that have used the CityBikes API as a data provider. However, CityBikes doesn't include timestamps, which makes it impossible to be sure that their data is in real time or not.") }}

{{ _("How do you predict the number of bikes/spaces at a station?") }}

{{ _("We use a machine learning. We have had the best results with the Random Forest algorithm.") }}

{{ _("Can you summarize how OpenBikes works under the hood?") }}

{{ _("We have a program that collects data from different cities. When the data is collected it is transformed so that all our data is uniform regardless of where it comes from. We run our machine learning algorithm on all the data we have collected, we do this once a week. We then connect our predictions to the map.") }}

{{ _("Why is the prediction button not on the map of each city?") }}

{{ _("Quite often the data that we collect does not have associated timestamps, hence it's impossible to verify if the data is in real time or not. Without a timestamp we cannot run our prediction models. Some other applications use the current time as a timestamp but that leads to a lot of problems and imprecision.") }}

{{ _("Can I help?") }}

{{ _("Of course! We are merely students who do similar side projects during our spare time. The code for the application is available on GitHub, a lot of it is documented and you can send us a mail in case you have questions or suggestions.") }}
